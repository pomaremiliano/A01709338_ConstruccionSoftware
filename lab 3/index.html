    Preguntas Lab 1
    ¿Cuál es la diferencia entre Internet y la World Wide Web?

    El internet se refiere a una red de conexiones que tienen las computadoras alrededor del mundo. El internet es el medio. Y la World Wide Web es un sistema que se utiliza para gestionar la información en internet utilizando direcciones url y protocolos como http. La red que funciona en internet.

    ¿Cuáles son las partes de un URL?

    protocolo de comunicación: comúnmente http y https. dominio: identificador de dirección de la computadora donde nos estamos conectando. path/directorio: muestra dónde está ubicada la fuente. variables: se pueden poner variables al final del url y darles valor con signos =?+. Se puede definir el query cuando buscas algo en google por ejemplo.

    ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

    Get: Se utiliza para pedir información de la fuente sin que esta esté alterada. Lo utiliza un navegador cada vez que ponemos una ruta, es mandar una petición http para obtener información. Todos los hipervínculos hacen peticiones tipo get. Head: Es como usar get pero solo para pedir la parte de “head” sin que regrese algo del “body”. Post: Se utiliza para mandar información a un servidor. Con https va con el head normal y el body cifrado. Put: Se utiliza para editar el contenido en la fuente o resource desde el body. Patch: se utiliza para aplicar modificaciones parciales o que van a corregir algo. Delete: al utilizarlo, se elimina el resource.

    ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

    El método get y post pueden utilizarse para mandar un formulario, pero es más recomendable utilizar post ya que get tiene limitaciones. Además de que Post manda lo que codifica a través del body encriptado y no aparece en la url.

    Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

    Indica una solicitud exitosa. En otras palabras, la página carga correctamente con texto, imágenes, etc. No ocurrió ningún error.

    ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

    Not Found. En algunos casos sí ya que puede ser que la página no existe porque el administrador modificó el url o lo transfirió, sin embargo la mayoría de las veces ocurre porque está mal escrito el url. Normalmente todos los códigos de 400 son responsabilidad o errores del cliente.

    ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

    Es un error muy general del servidor que se da por una falla en el código y si es responsabilidad de los desarrolladores para encontrar el código y solucionarlo. Normalmente los errores de 500, son errores de servidor.

    ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

    Los atributos depreciados no se recomiendan según los estándares de html5 ya que son viejos y están siendo reemplazados por atributos nuevos que cumplen con la misma funcionalidad. En html 4 se utilizan letras como “i” para itálicas y “b” para bold. Pero en html 5 se introdujo la forma de semántica en el texto, así que con solo usar palabras como “strong” se puede aplicar un bold a las palabras y además agregarle contexto para los demás lenguajes de programación con los que trabaja html.

    ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

    HTML5 fue una actualización que introdujo soporte multimedia y mayor compatibilidad con los navegadores. Se dice que con HTML4 las páginas se podían romper fácilmente debido a las funciones ya obsoletas que no tenían semántica y no podían comunicarse bien con otros lenguajes de programación. Así que yo diría que la mayor diferencia es la mejora en compatibilidad que tiene HTML5 sobre HTML4 al introducir nuevos atributos y capacidades.

    ¿Qué componentes de estructura y estilo tiene una tabla?

    Se utiliza “table” para definir la tabla. Se utiliza “tr” para las filas. Se le puede agregar un header con “th”. El contenido de la tabla se agrega con “td” dentro de las “tr”. Para agrandar una columna se utiliza “rowspan” y para hacer lo mismo con las columnas se utiliza “colspan” y se elige cuántas columnas o filas ocupará de espacio. Igual se pueden utilizar thead, tbody, y tfoot para agregar header body y pie de página a la tabla.

    Sobre el ciclo de vida y desarrollo de los sistemas de información:
    ¿Cuál es el ciclo de vida de los sistemas de información?

    Planificación, Diseño, desarrollo, pruebas, implementación, pruebas, mantenimiento, fin de la vida útil y migración a una actualización. Al final tiene que ver con un ciclo general que describe la concepción de los sistemas hasta cómo son tratados al terminar su vida útil.

    ¿Cuál es el ciclo de desarrollo de sistemas de información?

    Hay distintos tipos de desarrollo. Hay iterativos, tradicionales como los de cascada, ágiles (para el análisis, desarrollo y pruebas de lo más importante).
